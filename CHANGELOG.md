## How to Use This Changelog

### Guidelines for Contributors

- When making changes, update this file to document them
- Use these sections: Added, Changed, Deprecated, Removed, Fixed, Security
- Keep the Unreleased section at the top
- When releasing a new version, create a new section with the version and date
- Follow the format: `## [X.Y.Z] - YYYY-MM-DD`

### Version Format

- **Major** - Breaking changes
- **Minor** - New features (backwards compatible)
- **Patch** - Bug fixes and improvements

Example:
- 1.0.0 → 1.1.0 = new feature
- 1.0.0 → 2.0.0 = breaking change
- 1.0.0 → 1.0.1 = bug fix
